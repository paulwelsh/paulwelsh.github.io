eWeb ActiveX Control

This section details the eSellerate Embedded Web Store SDK provided by the eSellerate ActiveX control option for eSeller integration into Windows applications developed using Visual Basic or other COM-compatible tool. It documents the functions of the three ActiveX objects that can be used with this SDK option, which are: eWebSDK, eWebPrefillData and eWebResultData.

eWebSDK object functions

eWebSDK object functions enable everything from installing the Embedded Web Store Engine to software purchasing and activation. Some of these functions return result data, which can then be accessed through eWebResultData object functions. The eWebSDK object functions are:

  • eWeb_IsSystemCompatible
  • eWeb_InstallEngineFromPath
  • eWeb_Purchase
  • eWeb_PurchaseEx
  • eWeb_CheckForUpdate
  • eWeb_CheckForUpdateEx
  • eWeb_SilentCheckForUpdate
  • eWeb_WebStoreURL
  • eWeb_ActivateSerialNumber
  • eWeb_ManualActivateSerialNumber
  • eWeb_ValidateActivation
  • eWeb_DateActivation
  • eWeb_DeactivateSerialNumber
  • eWeb_ConfirmSerialNumber
  • eWeb_ValidateSerialNumber
  • eWeb_StandardizeSerialNumber
  • eWeb_Today

eWebPrefillData object functions

eWebPrefillData object functions enable prefilling data to simplify the purchase process for customers. Some examples of prefill capabilities include prefilling the customer's billing information, adding one or more SKUs to the shopping cart, and activating all SKUs purchased. The eWebPrefillData object functions are:

  • eWeb_AddPrefillData

eWebResultData object functions

eWebResultData object functions enable access to result data returned through Standard and Extended functions that invoke the Embedded Web Store Engine. The eWebResultData object functions are:

  • eWeb_GetOrderItemsByIndex
  • eWeb_GetResultDataValue
  • eWeb_IndexGetResultDataValue
Notes
  • The system requirements for this option are Windows XP or later, or Windows 2000 or later, and Internet Explorer 5.5 or later.
  • For a summary of previous and current function calls applicable to the eWeb ActiveX control, see New Functions Summary.
  • The ActiveX control option of the Embedded Web Store SDK for Windows is compatible with 64-bit Windows editions such as Windows XP Professional x64 Edition. This is not a 64-bit solution, but you can integrate it with your 32-bit applications to allow Embedded Web Store functionality on 64-bit Windows.
  • Beginning with the version 3.6.5 SDK, the recommended install locations for eWebControl365.dll is different for Vista than other Windows versions. The recommended location is the ProgramData\eSellerate directory on Vista, and the Common Files\eSellerate directory on pre-Vista Windows. As with previous SDK versions, you will need to register eWebControl365.dll after installing it.
  • Beginning with the version 3.6.5 SDK, the default install location for the eSellerate Embedded Web Store Engine (eWebClient.dll) is different for Vista than other Windows versions. The eWeb_InstallEngineFromPath function has been updated to install the engine in the ProgramData\eSellerate directory on Vista, and the Common Files\eSellerate directory on pre-Vista Windows.
  • Beginning with the version 3.6.5 SDK, it is no longer necessary to register the eSellerate Embedded Web Store Engine (eWebClient.dll). The eWeb_InstallEngineFromPath function has been updated for this change.
