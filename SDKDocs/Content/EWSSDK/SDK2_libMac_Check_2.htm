eWeb_CheckForUpdate

Checks for an available update for a given SKU.

Required Files

  • EWSEmbedded.framework (standard) or
    EWSEmbedded-GC.framework (garbage collecting)
    Included in the development project.

Definition

OSStatus eWeb_CheckForUpdate ( esellerID, SkuRefNum, previewID, layoutCertificate, trackingID, affiliateID, couponID, activationID, extraData, resultData )

esellerID – eSeller ID code

SkuRefNum – SKU RefNum Code

previewID – Preview Certificate string (optional)

layoutCertificate – Layout Test Certificate string (optional)

trackingID – tracking ID code (optional)

affiliateID – affiliate ID code (optional)

couponID – coupon ID code (optional)

activationID – Product Activation Set ID string (optional)

extraData – special serial number data string (optional)

resultData – pointer to memory that will be allocated and filled by the engine


See eSellerate Return Values for details on return values.

Parameter

Description

esellerID

String containing the Embedded Web Store ID, as found on the Sales Manager Embedded Web Store eSeller Summary page. Required.

SkuRefNum

String containing the SKU RefNum, as found on the Sales Manager Embedded Web Store SKU Attributes page. Required.

previewID

String containing the Preview Certificate, as found on the Sales Manager Embedded Web Store eSeller Summary page. Required to purchase in Preview mode. Pass NULL in Active mode. If the Preview Certificate passed here does not match the Preview Certificate on the Sales Manager Embedded Web Store eSeller Summary page, the purchase will be Active. This means that you can clear the Preview Certificate in the Sales Manager to activate the eSeller without having to recompile your application.

layoutCertificate

String containing the Layout Test Certificate, as found on the Settings tab of the Custom Layout Detail page. Allows testing of unpublished Custom Layouts. May be NULL. If the Layout Test Certificate passed here does not match the one on the Settings tab of the Custom Layout Detail page, the eSeller will use the published version of the Custom Layout. This means that you can clear the Layout Test Certificate in the Sales Manager to use the published layout without having to recompile your application.

trackingID

String containing special information that marks the function call so that it can be tracked in reports. May be NULL. See "tracking IDs" in the eSellerate User's GuidePublishers Help index.

affiliateID

String containing an affiliate ID that marks the function call so that it can be tracked in reports. May be NULL. For information on the eSellerate Affiliates program, see "eSellerate Affiliates: joining" in the eSellerate User's Guide index

couponID

String containing a coupon ID that enables coupon discounts on qualified purchases. May be NULL. See "coupons: creating" in the eSellerate User's GuidePublishers Help index.

activationID

String containing an activation ID for use in eSellerate Product Activation. May be NULL. Automatic activation occurs if the Embedded Web Store Engine completes the action and returns a unique serial number for use with the activationID (barring any activation errors). See "Product Activation: setting up" in the eSellerate User's GuidePublishers Help index.

extraData

String containing special additional information needed for serial number generation. Required for serial number type option Application Supplied and optional for Standard, Standard Enhanced and Armadillo. For all other serial number type options, pass NULL. See "serial numbers: Serial Number Type options" in the eSellerate User's GuidePublishers Help index.

resultData

Address of a declared variable of type char *. The variable may be initialized to NULL. Memory is allocated and filled when eWeb_CheckForUpdate is called. This memory must later be de-allocated by calling eWeb_DisposeResultData.
Parameters for eWeb_CheckForUpdate Function

Note: All strings passed into the engine are UTF-8 encoded cStrings.

OSStatus resultCode = eWeb_CheckForUpdate("STR9199856688", "SKU943434078", "NzIk6XXD", NULL, NULL, NULL, NULL, NULL, NULL, &resultData);
