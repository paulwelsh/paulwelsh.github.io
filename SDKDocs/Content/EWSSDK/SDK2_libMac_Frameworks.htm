eWeb Libraries for Macintosh

Two different versions of the eSellerate Embedded Web Store Engine are available for use with eWeb Libraries for Macintosh. The only difference between these engine versions is that one is always included within the application, while the other is typically installed in locations external to the application. Developers can use either or both of these engines to suit their needs.

Each engine version has its own SDK, and this documentation is divided into Embedded and Non-Embedded sections to show the differences.

The two engine versions are as follows:

Embedded Framework

Benefits:
  • Follows Apple's recommendation that custom frameworks intended for internal use should be installed inside the application that uses them
  • Eliminates permissions issues related to installing the framework externally
  • Application can be dragged to a local system and used without any additional steps for installing the framework
Limitations:
  • Works with applications only; other products must use the Non-Embedded Framework
  • Only one product can access the framework (no sharing allowed)
  • Does not support automatic engine updating

For more information, see eWeb Libraries for Macintosh — Embedded Framework.

Non-Embedded Framework

Benefits:
  • Works with products that are not applications, such as plug-ins
  • Installing the framework externally allows multiple products to share it
  • Supports automatic engine updating (when that functionality is enabled for the publisher)
Limitations:
  • Possible permissions issues related to installing the frameworks externally
  • Requires a separate installation step (typically, calling eWeb_InstallEngineFromPath)

For more information, see eWeb Libraries for Macintosh — Non-Embedded Framework.
