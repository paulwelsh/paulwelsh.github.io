eSellerate_DateActivation

For eSellerate Product Activation purposes, returns the date (expressed as the number of days since Jan. 1, 2000) of the last valid activation. This involves checking a hidden activation key on the computer.

Required Files

  • eSellerate.h
    Included in the development project.
  • eSellerateLibrary.lib(Carbon) or
    libeSellerate.a (Mach-O version)
    Included in the development project.
  • eSellerateEngine (Carbon)
    Located in the appropriate folder. It may be already placed here by another eSellerate-enabled application. The location depends on the platform:
    • For Mac OS 9.1 or later with CarbonLib 1.5 or later, place eSellerateEngine (Carbon) in System Folder:Application Support:MindVision
    • For Mac OS X, place eSellerateEngine (Carbon) in /Library/Application Support/MindVision

Definition

eSellerate_DaysSince2000 eSellerate_DateActivation ( publisherID, activationID, serialNumber )

eSellerate_InputString publisherID – Publisher ID string

eSellerate_InputString activationID – Product Activation Set ID string

eSellerate_InputString serialNumber – serial number string

Return Value

Description

eSellerate_NOENGINE

Returned if the eSellerateEngine cannot be found.

0

Returned for invalid activation, or for an activation key that does not contain a date. An activation key will contain a date if the associated activation ID was created after March 29, 2007. IDs created after that date will be at least 74 characters long.

n

Returned for a valid activation, where n is a date expressed as the number of days since Jan. 1, 2000.
Return Values for eSellerate_DateActivation Function
