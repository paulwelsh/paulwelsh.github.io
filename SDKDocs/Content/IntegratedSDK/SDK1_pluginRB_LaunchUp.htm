LaunchUpdate

Launches a standalone executable, such as an installer, or applies an Updater VISE file to a specified file and launches the resulting updated file.

Required Files

  • eSelleratePlugin.rbx
    Located in the REALbasic:Plugins folder.
Macintosh Targets
  • eSellerateEngine (Carbon)
    Located in the appropriate folder. It may be already placed here by another eSellerate-enabled application. The location depends on the platform:
    • For Mac OS 9.1 or later with CarbonLib 1.5 or later, place eSellerateEngine (Carbon) in System Folder:Application Support:MindVision
    • For Mac OS X, place eSellerateEngine (Carbon) in /Library/Application Support/MindVision
Windows Targets
  • eSellerateEngine.dll
    Located in the appropriate directory. It may be already placed here by another eSellerate-enabled application. The location depends on the platform:
    • For Windows Vista, place eSellerateEngine.dll in the ProgramData\eSellerate directory
    • For pre-Vista Windows versions, place eSellerateEngine.dll in the Windows directory

Definition

LaunchUpdate (

updaterFile as FolderItem, – location of an Updater VISE file to apply or an executable to launch

fileToUpdate as FolderItem – location of the file that an Updater VISE file is to be applied to

) as Integer

Return Value

Description

-2

Returned if eSellerateEngine cannot be found.

-1

Returned if updaterFile cannot be launched.

0

Returned if updaterFile does launch.
Return Values for LaunchUpdate Function

Parameter

Description

updaterFile

Location of either an Updater VISE file to apply to fileToUpdate or a standalone executable to launch. Required.

fileToUpdate

Location of the file to apply updaterFile to, when updaterFile is an Updater VISE file. If it is not, pass nil to launch a standalone executable, such as an installer.
Parameters for LaunchUpdate Function

Note: It may well be the case that a running application calls LaunchUpdate for the purpose of updating itself. In this situation, the application should promptly terminate itself after calling LaunchUpdate, since the application is then to be replaced.

Example

dim eSellerate as eSeller

dim resultCode as Integer

dim updaterFile as FolderItem

dim fileToUpdate as FolderItem

eSellerate = new eSeller

' ** TO DO: obtain updaterFile information and, if needed, fileToUpdate information **

resultCode = eSellerate.LaunchUpdate(updaterFile, fileToUpdate)

' ** TO DO: if fileToUpdate is the running application, exit application **
