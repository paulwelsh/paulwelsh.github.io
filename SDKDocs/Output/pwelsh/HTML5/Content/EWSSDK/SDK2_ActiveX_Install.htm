eWeb_InstallEngineFromPath

Installs the eSellerate Embedded Web Store Engine (eWebClient.dll) from the path specified to the proper location on the user's machine. Replaces an older version of the engine if one is present. See also Default Locations for eWebClient.dll.

When this function is called, you will pass it a path to the .dll. This file must exist somewhere else on the user's machine already. It is not included in the resources of the application, but would have to be included in the installer or somehow placed on the user's machine alongside the application. This will essentially take the .dll and place it in the proper eSellerate directory.

For information on installing the eSellerate Embedded Web Store Engine with an installer instead of the eWeb_InstallEngineFromPath function, see Installing eWebClient.dll and eWebControl365.dll.

Tip: For the best protection against engine tampering, you should compile your application with the latest eSellerate libraries, and place the latest eSellerate Embedded Web Store Engine where it will be available for installing from a specified path. Next, use the API call eWeb_InstallEngineFromPath immediately before any call that requires the engine. Calling eWeb_InstallEngineFromPath replaces an altered engine, if one is present.

Example

Dim esellerate As eWebSDKLib.eWebSDK

Dim result As eWebSDKLib.eWeb_ReturnValues

 

esellerate = CreateObject("eWebSDK.365")

result = esellerate.eWeb_InstallEngineFromPath("c:\path\to\engine\eWebClient.dll")
