Installing eWebClient.dll and eWebControl365.dll

The following files must be present on the target computer system to enable eSellerate functionality through the eWeb ActiveX control:

  • eSellerate Embedded Web Store Engine (eWebClient.dll)
  • eWebControl365.dll

Installing eWebClient.dll

The SDK offers the following option for installing the eWebClient.dll:

  • eWeb_InstallEngineFromPath copies the engine from the path specified to the proper location on the user's machine. See eWeb_InstallEngineFromPath.

This is the recommended option for handling engine placement. However, if you need a different approach, another option is to use an installer to install the engine, as described here.

Required steps for manually installing the eSellerate Embedded Web Store Engine are as follows:
  1. To determine whether the operating system will support the engine, check for the following:
    • Windows XP or later, or Windows 2000 or later

    • Microsoft Internet Explorer 5.5 or later

    If the check fails, launch a standard eSellerate Web Store to allow your customers to complete their purchases. Otherwise, continue with step 2.

  2. Check the eSellerate directory for an existing version of the engine. (See Default Locations for eWebClient.dll for the list of default locations.) If no engine exists in that location, or if the existing version is older than the one in your installer, continue with step 3. Otherwise, skip installation of the engine.
  3. Install the engine to the proper location (see Default Locations for eWebClient.dll), replacing the existing version if needed.

Installing eWebControl365.dll

In contrast to the eWebClient.dll, there is no SDK option for installing the eWebControl365.dll. Therefore, you will need to install and register the eWebControl365.dll manually.

The eWebControl365.dll can be installed and registered anywhere on the target system that the user can access. We recommend using the same install locations as the engine, which are as follows:

  • For Windows NT-based operating systems including Windows NT, Windows 2000 and Windows XP, use the Program Files\Common Files\eSellerate directory. A typical path is C:\Program Files\Common Files\eSellerate\eWebControl365.dll.
  • For Windows Vista, the default location is the ProgramData\eSellerate directory. A typical path is C:\ProgramData\eSellerate\eWebControl365.dll.
