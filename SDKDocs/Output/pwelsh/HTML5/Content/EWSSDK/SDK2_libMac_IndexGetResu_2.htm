eWeb_IndexGetResultDataValueLength

Retrieves the length (in bytes) of a result value given the combination of record or field name and index value.

Tip: This API can retrieve the length of result values for indexed eSellerate variables only. Indexed variables are designated as "Loop Only" on the variables page at http://publishers.esellerate.net/embeddedvar.aspx. To retrieve the length of a non-indexed variable, use eWeb_GetResultDataValueLength.

Required Files

  • EWSEmbedded.framework (standard) or
    EWSEmbedded-GC.framework (garbage collecting)
    Included in the development project.

Definition

OSStatus eWeb_IndexGetResultDataValueLength ( resultData, Record, Field, Index, Length )

resultData – results obtained from the Purchase or CheckForUpdate family of functions

Record – name of the record that contains the value whose length is being searched

Field – name of the field whose length you wish to identify

Index – zero-based index of the record/field instance

Length – length (in bytes) of the requested record and field


For a listing of all eSellerate variables that Embedded Web Store eSellers can pull from result data, use the following URL: http://publishers.esellerate.net/embeddedvar.aspx.

See eSellerate Return Values for details on return values.

Parameters for eWeb_IndexGetResultDataValueLength Function

Parameter

Description

resultData

Address of a declared variable of type char *. This should have already been filled by eWeb_Purchase or eWeb_CheckForUpdate.

Record

Name of the record that contains the value whose length is being searched. Required. An example of a valid record name follows:

Complete variable name: _CartItem.Quantity
Record name: _CartItem

Field

Name of the field whose length you wish to identify. Required. An example of a valid field name follows:

Complete variable name: _CartItem.Quantity
Field name: Quantity

Index

Zero-based index of the record/field instance. Required.

Length

Length (in bytes) of the requested record and field. Required.

Note: All strings passed into the engine are UTF-8 encoded cStrings.

#import<EWSEmbedded/EWSEmbedded.h>
