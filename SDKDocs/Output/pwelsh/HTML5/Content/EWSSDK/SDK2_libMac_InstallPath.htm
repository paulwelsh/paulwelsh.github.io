eWeb_InstallEngineFromPath

Copies the eSellerate Embedded Web Store Engine (EWSMac.framework/EWSMac-GC.framework) from the path specified to the proper location on the user's machine. This location will be /Library/Frameworks if the user has write access; otherwise the location will be ~/Library/Frameworks. Replaces an older version of the engine if one is present.

Before you call this function, you will need to place the engine somewhere that will make it available for copying. We recommend including it in your application's resources.

This function requires an absolute path to the engine. The engine that you reference in the path can be either the compressed version (EWSMacCompress.tar.gz/EWSMacCompress-GC.tar.gz) or the non-compressed version (EWSMac.framework/EWSMac-GC.framework). In either instance, the eWeb_InstallEngineFromPath function will install a non-compressed version of the engine to its proper location.

You should not link the engine into your application. The reason is that libEWS.a/libEWS-GC.a will load the engine dynamically when you call an Embedded Web Store invoking function such as eWeb_Purchase or eWeb_CheckForUpdate.

path

A path string that locates the engine to install. Must be an absolute path. Required.

Tip: For the best protection against engine tampering, you should compile your application with the latest eSellerate libraries, and include the latest eSellerate Embedded Web Store Engine as a resource. Next, use the API call eWeb_InstallEngineFromPath immediately before any call that requires the engine. Calling eWeb_InstallEngineFromPath replaces an altered engine, if one is present.
