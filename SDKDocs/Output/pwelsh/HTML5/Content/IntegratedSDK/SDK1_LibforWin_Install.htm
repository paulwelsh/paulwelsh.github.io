eSellerate_InstallEngine

Copies the eSellerateEngine from the application to its proper location, if one of the following conditions are true:

  • No engine is present.
  • The engine that is present is older than the one carried by the application. In this case, the function replaces the older engine.
  • The engine that is present has been altered. In this case, the function replaces the altered engine.

This approach requires that the eSellerateEngine is built into the application as a resource. While this makes the application file bigger, it allows the application to not depend on an installer or other strategy to ensure that the engine is in place.

Required Files

  • eSellerate.h
    Included in the development project.
  • eSellerateLibrary.lib
    Included in the development project.
  • eSellerateEngine.dll
    Included in the development project and built into the application as a resource of resource type DLL and resource ID 1.

Definition

eSellerate_ErrorCode eSellerate_InstallEngine ()

Return Values for eSellerate_InstallEngine Function

Return Value

Description

eSellerate_SUCCESS

Returned if the eSellerateEngine is already properly installed, or if this function installed the engine.

eSellerate_FAILURE

Returned if the eSellerateEngine is not already properly installed, and this function could not install the engine (perhaps because the eSellerateEngine was not built into the application as a resource).

Example

#include "eSellerate.h"

eSellerate_ErrorCode resultCode = eSellerate_InstallEngine();

/* TO DO: handle error, or use the eSellerateEngine */

Tip: For the best protection against engine tampering, you should compile your application with the latest eSellerate libraries, and include the latest eSellerateEngine as a resource. Next, use the API call eSellerate_InstallEngine immediately before any call that requires the eSellerateEngine. Calling eSellerate_InstallEngine replaces an altered eSellerateEngine, if one is present.
