Integrated eSeller: CheckForUpdate Redirect

The CheckForUpdate Redirect feature allows an application to use the same eSeller RefNum for purchasing and checking for updates. This is useful because only one Integrated eSeller is required for initial setup. When you're ready to release an update, you can create a second Integrated eSeller for that purpose, and redirect to it from the original eSeller.

Tip: For an example of how to handle updates throughout several product releases, see CheckForUpdate Redirect Example.)

To set the CheckForUpdate Redirect:

  1. In the CheckForUpdate Redirect dialog box, click a radio button to select one of the following options:

    CheckForUpdate Redirect Options
    Option Description
    Use current eSeller, no update available This is the appropriate setting for the first time you release your application as an Integrated eSeller, and any other time that no update is available. When a customer uses your software to check for a newer version, the CheckForUpdate Message specified for this eSeller will display. (See Integrated eSeller: Software Delivery Wizard.) After reading your message, the customer will need to click OK, which will return control to your application.
    Use current eSeller, update available Use this setting if an update is available, and this Integrated eSeller will provide the update. When a customer uses your application to check for a newer version, the CheckForUpdate Message specified for this eSeller will display. (See Integrated eSeller: Software Delivery Wizard.) After reading your message, the customer will have the option to receive the newer version, which could be a free upgrade or a purchased upgrade.
    Redirect to different eSeller Use this setting if an update is available, and a different Integrated eSeller will provide the update. Make your eSeller selection from the list provided.
  2. Click Save Changes.

Tip: For a look at all the CheckForUpdate Redirects set up for your Integrated eSellers, use the Integrated eSeller CheckForUpdate Redirects report, which is available in Admin Reports.
