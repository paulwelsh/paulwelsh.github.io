Creating the SKU and Setting Its Basic Properties

To create a SKU and set its Basic properties:
  1. Click the SKUs tab. The SKU List page appears.
  2. Click New. The SKU Detail page appears.

  3. Fill out the SKU Details page as described here.

    In the following table, a check mark for "eSellers" indicates that the item is presented to customers when they purchase through one of your eSellers. A check mark for "Partner Data Feed" indicates that the item is included in our partner data feed that goes to the Digital River Try and Buy Network. To learn more about the data feed, please see http://www.esellerate.net/try_buy_network.asp.

    The following items are required entries: SKU ID, SKU Name, ECCN and Country of Origin. All other items on the page are optional.

    SKU Basic Tab Options
    Label Description eSellers Partner Data Feed
    SKU ID SKU ID will be visible to your customers on the receipt page, and in the Order Confirmation e-mail. The ID appears immediately after the SKU name, enclosed by either parentheses or brackets. For example, a SKU named "Widget" with a SKU ID of "MyID" would display as: Widget (MyID)

    If desired, enter a SKU ID of up to 50 characters. Otherwise, you can accept the default, which is the same value as the SKU RefNum (a non-editable code used for internal tracking).     		X X
    SKU Name Enter a descriptive SKU Name. The name will appear with the SKU when you list it in eSellers. X X
    Price (USD) In the Price (USD) field, enter the price (in U.S. dollars) that you want to present for this SKU when you list it in eSellers. The price must be either $0.00 or $4.95 or above.

    Note: To set Native Currency Pricing (NCP) for this SKU, click Save, and then click the Edit Pricing icon.    		 X X
    SKU Download If you choose, the eSellerate system can download a specified file when it delivers this SKU. You set the default download file here. (You can also override this setting with an individual eSeller.)

    To select a download file, use the SKU Download list. If you need to edit or create a download file before you make your selection, click Save, and then click the Edit Downloads icon.

    Note: To delete a download file, use the Downloads editor (available in the References section of the eSellerate Sales Manager).    		 X X
    Serial Number If you select a serial number set here, eSellerate will issue serial numbers for this SKU when it is purchased. You can then use the serial numbers any way you choose. Typical uses include registering products and protecting products against software piracy.

    To select a serial number set, use the Serial Number list. If you need to edit or create a serial number set before you make your selection, click Save, and then click the Edit Serial Numbers icon.     		X X
    Short Description and "More Info" Text Complete the Short Description and "More Info" Text fields to provide prospective customers information about the SKU. This information will appear with the SKU when you list it in eSellers. (The exception is the primary SKU in an Integrated eSeller, which the user will have already selected.)

    The maximum number of characters is 500 for Short Description and 4,000 for "More Info" Text. (When used in the partner data feed, the strings are truncated if necessary, to the maximum of 128 characters for Short Description and 884 for "More Info" Text.)

    For SKUs you offer through Web Store and Embedded Web Store eSellers, the Short Description and "More Info" Text fields will accept any of the valid HTML tags listed at http://www.webmonkey.com/2010/02/html_cheatsheet/.

    In addition, the HTML representations listed under "Name Code" and "Number Code" at http://www.webmonkey.com/2010/02/special_characters/ are valid anywhere the eSellerate system supports HTML.

    Note: When you enter special characters in the Short Description and "More Info" Text fields, be sure to use valid HTML tags. You will encounter display problems if you enter special characters by methods other than valid HTML tags. For example, you should not enter special characters by copying and pasting symbols from Microsoft Word's symbol set, or by using keystrokes such as Option+2 for a trademark, or Option+8 for a bullet (Macintosh).

    Note: Although Integrated eSellers do not support the use of HTML tags in the Short Description and "More Info" Text fields, there is a related option that allows any eSeller to display a basic web page in place of the "More Info" Text (with some restrictions). This option is available only when the SKU is displayed as a cross-sell item. For more information, see "Cross-Sell SKU Options" on page 1.    		 X X
    Comment The comment is for internal identification purposes only, and will not be visible to your customers. If desired, enter a Comment that applies to this SKU.    
    ECCN and Country of Origin

    ECCN and Country of Origin are required entries. Their purpose is as follows.

    eSellerate adheres to U.S. export controls and requires certain export control information for all SKUs. More information about export controls is available on these U.S. government web pages: Export Control Basics and Policies and Regulations. For information about your obligations to eSellerate in connection with export, please see this page.

    Use the ECCN list to select the ECCN that describes this SKU.

    Use the Country of Origin list to select the country where the SKU was manufactured or produced.

    If the ECCN you selected for this SKU requires you to provide additional details, click Show Additional Options. Then complete any of the following items that apply:

    • License Exception – See License Exception Code.
    • CCATS – See CCATS Code.
    • Harmonize Code – See Schedule B/HTS Code.
    		    
    Image Type You can upload the following image types for this SKU: Boxshot, Screenshot 1, Screenshot 2, Screenshot 3 and Screenshot 4.

    Boxshot is the image that will appear with the SKU when you list it in eSellers. (The exception is the primary SKU in an Integrated eSeller, which the customer will have already selected.)

    The four different Screenshots are for use in the partner data feed only.

    To specify the image that you want to upload, make your selection from the Image Type list. (Then follow the instructions under "Image Path" to upload the image.)    		 X X
    Image Path This is where you upload an image after selecting the image type, such as Boxshot or Screenshot 1.

    Click Browse (or Choose File if you're using Chrome or Safari) and then select the desired image. Next, click Save, which will start the upload.

    Note: Boxshots and Screenshots that you upload to the Sales Manager must be in PNG, JPG, GIF or BMP formats. For best results, we recommend using 8-bit graphics with a resolution of 72 pixels per inch.

    The maximum dimensions for Boxshots are 220 x 220 pixels.

    The minimum and maximum dimensions for Screenshots are 640 x 480 pixels and 2048 x 2048 pixels, respectively.

    You should not save these files in MacBinary format, even if you're uploading them from a Macintosh.     		  X
    Image Caption This option is available only when Image Type = Screenshot <x>. In the Image Caption field, enter the caption that you would like to show for the selected image. The maximum number of characters is 128.   X
    Platform Platform is a value that you can use to identify this SKU in searches. The platform will not be visible to your customers.

    To select a platform, use the Platform list.    		    
    Operating System This option is available only when Platform = Windows, Windows CE or Macintosh. Select the check box for each operating system that you want to list for this SKU.   X
    Category Category is a value that you can use to identify this SKU in searches. The category will not be visible to your customers.

    To select a category, use the Category list. If you need to edit or create a category before you make your selection, click Save, and then click the Edit Categories icon.    		    
    Version We recommend that you always complete the Version field for SKUs that you are including in the partner data feed.   X
    Release Date If you want to show a release date for this SKU, enter it in the Release Date field. Use the format MM/DD/YYYY.   X
    Icon Image Path Click Browse (or Choose File if you're using Chrome or Safari) and then select the icon that you want to upload for your program. For example, you could upload an Apple Icon Image (.icns) file here.

    The upload will begin when you save your changes.    		   X
    Language If you want to show that your product is localized for certain languages, you can select the languages from the Languages list. To select consecutive languages, hold down the Shift key and click on the desired items. To select non-consecutive languages, hold down the Control key (Windows) or the Command key (Macintosh) and click on the desired items.   X
    Appcast URL If there is an appcast that will deliver updates and release notes for this SKU, you can include it in the partner data feed. Enter the URL in the Appcast URL field. The maximum number of characters is 256.

    Note: Appcasts rely on RSS feeds, so they are not limited to specific platforms. You can create your own appcasts by using the "enclosure" feature of RSS 2.0, or you can use third-party developer tools for that purpose. An example of an appcasting option for Macintosh is the Sparkle framework.    		   X
    Bundle ID This option is available only when Platform = Macintosh. Enter the Bundle ID exactly as it appears in the Info.plist file of your application. Bundle ID is case-sensitive, and the maximum number of characters is 200.   X
    Bundle Type This option is available only when Platform = Macintosh. Use the Bundle Type list to select the type for your application bundle. Your choices are Application, Preference Pane, Screen Saver and Widget.   X
    Release Notes To provide release information for this SKU, enter the desired text (up to 1,024 characters) in the Release Notes field. This field will accept any of the valid HTML tags listed at http://www.webmonkey.com/2010/02/html_cheatsheet/.   X
  4. Click Save. Notice that four tabs now display: Basic, Advanced, Bundle and Usage. The section that you just completed was the Basic tab.
