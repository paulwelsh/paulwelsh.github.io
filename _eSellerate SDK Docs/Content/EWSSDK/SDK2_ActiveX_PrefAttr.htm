eWeb_AddPrefillData

Allocates memory and fills it with prefill data. This function must be called once for every name/value pair you wish to add to prefill data.

Required Files

  • eWebControl365.dll
    Registered and typically located in the eSellerate directory noted at Installing eWebControl365.dll (the recommended location). It may be already placed here by another eSellerate-enabled application.
  • eWebClient.dll
    Located in the eSellerate directory noted at Default Locations for eWebClient.dll. It may be already be placed here by another eSellerate-enabled application.

Definition

eWeb_AddPrefillData ( name, value )

name – name of the eSellerate prefill variable being set

value – value being given to the eSellerate prefill variable


See eWebPrefillData Attributes for information on attributes that can be prefilled directly in the eWebAddPrefillData class without having to call eWeb_AddPrefillData.

For a listing of all eSellerate variables that Embedded Web Store eSellers can prefill or pull from result data, use the following URL: http://publishers.esellerate.net/embeddedvar.aspx.

See eSellerate Return Values for details on return values.

Parameter

Description

name

Name of the eSellerate prefill variable being set. Required. Must include a zero-based index if the variable is designated as "Loop Only" on the variables page at http://publishers.esellerate.net/embeddedvar.aspx. For example, use "_CartItem0.SkuRefNum" to set a SKURefNum value for the first cart item.

Be sure to note that when you prefill the _CartItem.SerialNumberExtraData variable, you must also prefill the corresponding _CartItem.SkuRefNum variable. For example, if you prefill _CartItem0.SerialNumberExtraData, you must also prefill _CartItem0.SkuRefnum before using the prefill data in a function such as eWeb_PurchaseEx. The _CartItem.SkuRefNum value specifies the SKU that should use the additional serial number information provided by _CartItem.SerialNumberExtraData.

value

Value being given to the eSellerate prefill variable. Required.
Parameters for eWeb_AddPrefillData Function

Example

Dim prefillData As eWebSDKLib.eWebPrefillData

Dim result As eWebSDKLib.eWeb_ReturnValues

 

prefillData = CreateObject("eWebPrefillData.365") 'Create the Prefill Data

 

' EXAMPLE ON HOW TO PREFILL SKU AND ACTIVATIONID

result = prefillData.eWeb_AddPrefillData("_CartItem0.SkuRefNum", "STR9199856688")

result = prefillData.eWeb_AddPrefillData("_CartItem0.ActivationID", "ACT123456789:QTNCLL-AVKJ-A73C62-EXPN-ECEBTK-89U5R0-VN2D-FCDW7J-UVTA-B1PHCA")

' PREFILL ANY OTHER ITEMS HERE

 

prefillData = Nothing

eWebPrefillData Attributes

The following attributes can be prefilled directly in the eWebResultData class after
creation of the eWebResultData object, without having to call eWeb_AddPrefillData:

  • BillingFirstName
  • BillingLastName
  • BillingAddress1
  • BillingAddress2
  • BillingCity
  • BillingState
  • BillingZip
  • BillingCountry
  • BillingEmail
  • BillingPhone
  • BillingCompany

Both lines in the example code below prefill the shopper's billing first name with "John." The first line prefills the data directly, and the second uses eWeb_AddPrefillData.

prefillData.BillingFirstName = "John"

result = prefillData.eWeb_AddPrefillData("_Shopper.BillingFirstName", "John")
