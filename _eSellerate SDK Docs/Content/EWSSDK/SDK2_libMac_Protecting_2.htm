Protecting Mach-O Applications from Hackers

If you integrate eSellerate functionality into your Mach-O application, you should use the "strip" command from the command-line tool (Terminal.app) to remove any debug symbols in your final executable output. (The command is only available if Apple's developer tools are installed.) Stripping the debug symbols will help prevent hackers from reading eSellerate code that could compromise your application's security, such as functions that use eSellerate Product Activation.

For more information on the "strip" command, see its man page within the Terminal utility.
