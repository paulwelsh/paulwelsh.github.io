eSellerate Libraries for Windows

Note: The eSellerate (Integrated) SDK is a legacy option that was superseded by the Embedded Web Store SDK. It is not recommended for use with new projects. The Embedded Web Store SDK provides many advanced features that are not available with the legacy option, including fully customizable stores and universal binary support for Macs.

This section details the eSellerate SDK provided by the static library option for eSeller integration into Windows applications developed using Visual C++. It documents the functions:

  • eSellerate_InstallEngine
  • eSellerate_Purchase
  • eSellerate_CheckForUpdate
  • eSellerate_SilentCheckForUpdate
  • eSellerate_Reinstall
  • eSellerate_IndexSerialNumberData
  • eSellerate_IndexFileLocationData
  • eSellerate_DeleteResultData
  • eSellerate_LaunchUpdate
  • eSellerate_WebStoreURL
  • eSellerate_ActivateSerialNumber
  • eSellerate_ManualActivateSerialNumber
  • eSellerate_ValidateActivation
  • eSellerate_DateActivation
  • eSellerate_DeactivateSerialNumber
  • eSellerate_ConfirmSerialNumber
  • eSellerate_ValidateSerialNumber
  • eSellerate_StandardizeSerialNumber
  • eSellerate_Today

For a summary of previous and current function calls applicable to eSellerate libraries for Windows, see New Functions Summary.

Tip: Beginning with the version 3.6.5 SDK, the default install location for the eSellerateEngine is different for Vista than other Windows versions. The eSellerate_InstallEngine function has been updated to install the engine in the ProgramData\eSellerate directory on Vista, and the Windows directory on pre-Vista Windows.
