eSellerate Plug-in for REALbasic

Note: The eSellerate (Integrated) SDK is a legacy option that was superseded by the Embedded Web Store SDK. It is not recommended for use with new projects. The Embedded Web Store SDK provides many advanced features that are not available with the legacy option, including fully customizable stores and universal binary support for Macs.

Note: For Macintosh applications — This is not a universal binary solution. For universal binary support and access to many other advanced features, you can use the eSellerate Embedded Web Store SDK. See eSellerate Embedded Web Store SDK.

Note: This note applies to eSeller integration into Windows applications only. Beginning with the version 3.6.5 SDK, the recommended install location for the eSellerateEngine is different for Vista than other Windows versions. The recommended location is the ProgramData\eSellerate directory on Vista, and the Windows directory on pre-Vista Windows.

Note: Beginning with the version 3.6.5 SDK, the following function calls require passing an empty parameter: InstallEngine, GetSerialNumberCount, GetFileLocationCount and GetShoppingCartCount. For example, "InstallEngine as Integer" became "InstallEngine () as Integer" in version 3.6.5.

Note: The version 3.6.5 introduced two notable changes for ValidateSerialNumber: publisherKey became a required parameter, and the return value for an invalid serialNumber changed from -1 to 0.
