eWeb_Purchase

Invokes the Embedded Web Store Engine to perform a purchase.

Required Files

  • EWSLib.h
    Included in the development project.
  • libEWS.a (standard) or
    libEWS-GC.a (garbage collecting)
    Included in the development project.
  • EWSMac.framework (standard) or
    EWSMac-GC.framework (garbage collecting)
    Typically located in the /Library/Frameworks folder, but in some cases will be located in the ~/Library/Frameworks folder. It may be already placed in one or both of these locations by another eSellerate-enabled application.

Definition

OSStatus eWeb_Purchase ( esellerID, SkuRefNum, previewID, layoutCertificate, trackingID, affiliateID, couponID, activationID, extraData, resultData )

esellerID – eSeller ID code

SkuRefNum – SKU RefNum code (optional)

previewID – Preview Certificate string (optional)

layoutCertificate – Layout Test Certificate string (optional)

trackingID – tracking ID code (optional)

affiliateID – affiliate ID code (optional)

couponID – coupon ID code (optional)

activationID – Product Activation Set ID string (optional)

extraData – special serial number data string (optional)

resultData – pointer to memory that will be allocated and filled by the eSellerate Embedded Web Store Engine


See eSellerate Return Values for details on return values.

Parameters for eWeb_Purchase Function

Parameter

Description

esellerID

String containing the Embedded Web Store ID, as found on the Sales Manager Embedded Web Store eSeller Summary page. Required.

SkuRefNum

String containing the SKU RefNum, as found on the Sales Manager Embedded Web Store SKU Attributes page. Causes the Embedded Web Store to add the referenced SKU to the shopping cart and display the cart page. May be NULL, in which case the Embedded Web Store will display the catalog page. For information on assigning the store layout's cart and catalog pages, see "Custom Layouts: Files tab" in the eSellerate Publishers Help index.

previewID

String containing the Preview Certificate, as found on the Sales Manager Embedded Web Store eSeller Summary page. Required to purchase in Preview mode. Pass NULL in Active mode. If the Preview Certificate passed here does not match the Preview Certificate on the Sales Manager Embedded Web Store eSeller Summary page, the purchase will be Active. This means that you can clear the Preview Certificate in the Sales Manager to activate the eSeller without having to recompile your application.

layoutCertificate

String containing the Layout Test Certificate, as found on the Settings tab of the Custom Layout Detail page. Allows testing of unpublished Custom Layouts. May be NULL. If the Layout Test Certificate passed here does not match the one on the Settings tab of the Custom Layout Detail page, the eSeller will use the published version of the Custom Layout. This means that you can clear the Layout Test Certificate in the Sales Manager to use the published layout without having to recompile your application.

trackingID

String containing special information that marks the function call so that it can be tracked in reports. May be NULL. See "tracking IDs" in the eSellerate Publishers Help index.

affiliateID

String containing an affiliate ID that marks the function call so that it can be tracked in reports. May be NULL. For information on the eSellerate Affiliates program, see "eSellerate Affiliates: joining" in the eSellerate Publishers Help index.

couponID

String containing a coupon ID that enables coupon discounts on qualified purchases. May be NULL. See "coupons: creating" in the eSellerate Publishers Help index.

activationID

String containing an activation ID for use in eSellerate Product Activation. May be NULL. Automatic activation occurs if the Embedded Web Store Engine completes the action and returns a unique serial number for use with the activationID (barring any activation errors). See "Product Activation: setting up" in the eSellerate Publishers Help index.

When you pass a non-NULL value for the activationID parameter, you must also pass a non-NULL value for the SkuRefNum parameter. The SkuRefNum specifies the SKU whose serial number should be activated according to the rules set by activationID.

extraData

String containing special additional information needed for serial number generation. Required for serial number type option Application Supplied and optional for Standard, Standard Enhanced and Armadillo. For all other serial number type options, pass NULL. See "serial numbers: Serial Number Type options" in the eSellerate Publishers Help index.

When you pass a non-NULL value for the extraData parameter, you must also pass a non-NULL value for the SkuRefNum parameter. The SkuRefNum specifies the SKU that should use the additional serial number information provided by extraData.

resultData

Address of a declared variable of type char *. The variable may be initialized to NULL. Memory is allocated and filled when eWeb_Purchase is called. This memory must later be de-allocated by calling eWeb_DisposeResultData.

Note: All strings passed into the engine are UTF-8 encoded cStrings.

Example

#include "EWSLib.h"

char *resultData = NULL;

OSStatus resultCode = eWeb_Purchase("STR9199856688", "SKU862610260", "NzIk6XXD", NULL, NULL, NULL, NULL, NULL, NULL, &resultData);
