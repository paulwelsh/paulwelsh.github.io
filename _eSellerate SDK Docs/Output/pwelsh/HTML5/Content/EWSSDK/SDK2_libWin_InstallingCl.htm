Installing eWebClient.dll

The eWeb libraries for Windows offer two primary options for installing the eSellerate Embedded Web Store Engine (eWebClient.dll) to enable eSellerate functionality:

  • eWeb_InstallEngineFromResource copies the engine from resources specified to the proper location on the user's machine. See eWeb_InstallEngineFromResource.
  • eWeb_InstallEngineFromPath copies the engine from the path specified to the proper location on the user's machine. See eWeb_InstallEngineFromPath.

These API calls are the recommended options for handling engine placement. However, if you need a different approach, a third option is to use an installer to install the engine, as described here.

Required steps for manually installing the eSellerate Embedded Web Store Engine are as follows:
  1. To determine whether the operating system will support the engine, check for the following:
    • Windows XP or later, or Windows 2000 or later

    • Microsoft Internet Explorer 5.5 or later

    If the check fails, launch a standard eSellerate Web Store to allow your customers to complete their purchases. Otherwise, continue with step 2.

  2. Check the eSellerate directory for an existing version of the engine. (See Default Locations for eWebClient.dll for the list of default locations.) If no engine exists in that location, or if the existing version is older than the one in your installer, continue with step 3. Otherwise, skip installation of the engine.
  3. Install the engine to the proper location (see Default Locations for eWebClient.dll), replacing the existing version if needed.
