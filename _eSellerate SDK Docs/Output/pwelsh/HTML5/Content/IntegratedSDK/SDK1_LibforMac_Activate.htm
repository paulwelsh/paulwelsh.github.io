eSellerate_ActivateSerialNumber

For eSellerate Product Activation purposes, confirms that a serial number originated from eSellerate and associates that serial number with the unique hardware identification of the computer in use.

Required Files

  • eSellerate.h
    Included in the development project.
  • eSellerateLibrary.lib(Carbon) or
    libeSellerate.a (Mach-O version)
    Included in the development project.
  • eSellerateEngine (Carbon)
    Located in the appropriate folder. It may be already placed here by another eSellerate-enabled application. The location depends on the platform:
    • For Mac OS 9.1 or later with CarbonLib 1.5 or later, place eSellerateEngine (Carbon) in System Folder:Application Support:MindVision
    • For Mac OS X, place eSellerateEngine (Carbon) in /Library/Application Support/MindVision

Definition

eSellerate_ErrorCode eSellerate_ActivateSerialNumber ( publisherID, activationID, serialNumber, askToConnect )

eSellerate_InputString publisherID – Publisher ID string

eSellerate_InputString activationID – Product Activation Set ID string

eSellerate_InputString serialNumber – serial number string

eSellerate_TrueOrFalse askToConnect – ask to connect when no active connection is found, and optionally display Manual Product Activation Wizard if connection attempt fails

Return Values for eSellerate_ActivateSerialNumber Function

Return Value

Description

eSellerate_NOENGINE

Returned if the eSellerateEngine cannot be found.

eSellerate_FAILURE

Returned if publisherID, activationID or serialNumber is nill or if the eSellerateEngine is unable to connect to the Internet. When no active connection to the Internet is found, the engine will return eSellerate_FAILURE if the value of the askToConnect parameter is false (zero) or if the value is true (non-zero) and the user cancels the dialog that asks for permission to connect. In addition, the engine will return eSellerate_FAILURE if the Manual Product Activation Wizard is displayed, and the user cancels.

eSellerate_SUCCESS

Returned if the eSellerateEngine completes the action.

-25000

Returned if activation fails because activationID was unknown.

-25001

Returned if activation fails because serialNumber was unknown.

-25003

Returned if activation fails because serialNumber is blacklisted.

-25004

Returned if activation fails because the product was returned.

-25005

Returned if activation fails because the activation limit was met.

-25009

Returned if activation fails because serialNumber is already associated with another order.

-25012

Returned if eSellerate denies the activation request.
Parameters for eSellerate_ActivateSerialNumber Function

Parameter

Description

publisherID

String containing the Publisher ID, as found on the Sales Manager Home page. Required.

activationID

String containing an activation ID for use in eSellerate Product Activation. Required. See "Product Activation: setting up" in the eSellerate Publishers Help index.

serialNumber

String containing the serial number. Required.

askToConnect

Logical value that controls whether a dialog is displayed asking the user for permission to connect to the Internet when no active connection is found. Also controls whether the Manual Product Activation Wizard is displayed after the user grants permission to connect to the Internet but no connection can be made. Pass a true (non-zero other than 2) value to enable display of both the connection dialog and the wizard. Pass a value of 2 to enable display of the connection dialog but not the wizard. Pass false (zero) to disable display of both the connection dialog and the wizard.

Note: Strings are handled as length-prepended Pascal strings.

Caution: The eSellerate_ActivateSerialNumber function cannot detect an active Internet connection on computers that are using an AirPort wireless network card. As a result, unless the function call prompts the user for permission to connect, the return value will be eSellerate_FAILURE. You can work around this limitation by passing a true (non-zero) value for the askToConnect parameter, which will enable display of the connection dialog.

Caution: Serial numbers used in Product Activation are case sensitive. This restriction is typically not an issue for SDK functions that handle serial numbers without user input, but it can be an issue if you use the eSellerate_ActivateSerialNumber function to allow customers to activate software purchased through a web store. To help prevent -25001 errors in this instance, you can configure your source code to correct any case discrepancies before passing a serialNumber parameter for the eSellerate_ActivateSerialNumber function. Also, in the prompt for your customers to enter the serial number, you can include language such as: "Enter the serial number exactly as shown on your receipt, including case (uppercase or lowercase)."
